## 원인 정리

### 1) 숙제 1/2 (정상 동작에 가까움)

- 집계: `COMPLETED` 숙제만, `due_at`이 보고서 구간 안인 것.
- 강학생 예시 (배정 5/26 15:45):
  - 5/26 15:35 마감 → **제외** (배정 전)
  - 5/26 18:00 → 미제출 → 분모 +1
  - 5/27 14:00 → 제출 → 분자 +1  
  → **2건 중 1건 = 1/2**
- 운영: 점수·제출은 **완료 처리 없이** 저장 가능, **보고서에는 `COMPLETED`만** 포함.

### 2) 클리닉 미집계 (버그)

- 기존: `clinic_slot.week_start_date`(월요일)만으로 구간 필터.
- 5/27(수) 클리닉 → `week_start_date = 5/25`, 구간 시작일(배정일) `5/26` → **제외**.
- 예약·출석 데이터는 있으나 집계 0건.

### 3) 종합 70점 vs 테스트 막대 100%

- 종합 30%: `percentile_rank` 사용 (상대 성적).
- UI 막대: `upper_rank_pct`(상위 %) 사용 → **역방향**이라 100%로 보이면 종합 기여는 0%에 가깝게 계산됨.
- 예: 숙제·클리닉 100% + 테스트 percentile 0 → `40+30+0 = 70`.

### 4) ZIP 다운로드 WARN

- `Content-Disposition`에 한글 파일명 → Tomcat ISO-8859-1 인코딩 실패 WARN (본문은 정상).

### 5) PDF 생성 실패 (일시)

- PDFBox 3 `setStrokingColor(220,220,220)` → 0~1 범위 위반. `0.86f` 등으로 수정.
