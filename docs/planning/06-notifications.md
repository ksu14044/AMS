# 06. 알림 시스템

학생·학부모 🔔 — **미완료 건수** 뱃지 (읽음과 무관).

## v3.0 원칙 ([§16](./DECISIONS.md#16-알림-완료-기준))

| 원칙 | 내용 |
|------|------|
| 발송 | 이벤트당 **1회** |
| 소멸 | 과제형은 **완료** 시만. **읽음 ≠ 소멸** |
| 뱃지 | `pending-count` = 미완료 숙제·테스트·영상·클리닉 합계 |
| D-1 | 숙제 **폐기** · 테스트 `test_at` **전날** 1회 |

## 완료 조건

| 유형 | 완료(소멸) |
|------|------------|
| 숙제 | **점수 입력** |
| 테스트·재시험 | **점수 입력** |
| 클리닉 | **결과 입력** |
| 영상 | **인증 사진 제출** (대상 학생) |
| 공지·배정·보고서 | 읽음 시 목록 정리 |

## 알림 유형

| 유형 | 트리거 | 수신 |
|------|--------|------|
| 공지 등록 | 반 공지 | 학생·학부모 |
| 숙제·테스트 등록 | 수업기록 귀속 등록 | **대상 학생** |
| 재시험 일정 | 일시 등록 | 재시험 대상 |
| 클리닉 확정 | 토 23:00 잠금 | 예약 학생 |
| 클리닉 변경 | 슬롯 수정 | 해당 학생 |
| 결과·보고서 | 입력·생성 | 학생·학부모 |

~~숙제 D-1~~ — 제거. 테스트 전날 알림(`test_at`) 유지.

## 학생 홈·알림

- `GET /me/pending-tasks` — 미완료 목록 (`accessible_from` 필터, 클리닉 제외)
- 알림 요약: "아직 완료하지 않은 과제가 N건" → 미완료 목록 화면

## 구현

- `notification.status`: ACTIVE | DISMISSED
- 과제형 알림: 완료 시 DISMISSED · 정보형: 읽음 시 DISMISSED
- 숙제 D-1 스케줄 제거 · 테스트 전날 스케줄 유지
