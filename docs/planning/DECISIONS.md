## §1. 역할 7종 (교직원 가입)

### 확정 — 7종 enum

| 코드 | 표시명 | 비고 |
|------|--------|------|
| `TEACHER_KO` | 국어 선생님 | 담임 가능, 과목 KO |
| `TEACHER_EN` | 영어 선생님 | 과목 EN |
| `TEACHER_MATH` | 수학 선생님 | 과목 MATH |
| `STAFF_OFFICE` | 행정 | 학원 전체 보조 |
| `ASSISTANT_KO` | 국어 조교 | |
| `ASSISTANT_EN` | 영어 조교 | |
| `ASSISTANT_MATH` | 수학 조교 | |

**원장:** `ACADEMY_ADMIN` — 7종 외, 승인 권한.

**학생:** `STUDENT` — 별도 가입 플로우.

**학부모:** `PARENT` — 학생과 동일 가입·승인 플로우 ([§19](#19-학부모-계정)).

## §3. 「출석」 (학생 목록 탭)

## §7. 영상 인증 — 가산 반영

| 지표 | 계산 |
|------|------|
| **종합 달성률(게이지)** | 숙제 완료률×0.4 + 클리닉 출석률×0.3 + 테스트 **점수 %** 평균×0.3 ([§13](#13-테스트-석차--게이지)) |
| **영상 인증률** | 별도 % · 보고서 항목 (종합 가산 없음) |

## §8. 숙제 [제출 완료] (v2.0, **v3.0 §11로 대체**)

v3.0: 교직원만 정오표·점수 입력. 완료 = 점수 저장. 학생 직접 입력 없음.

## §9. 테스트 점수 — 종합 게이지 (v2.0, **v3.0 §13으로 대체**)

v2.0의 `percentile_rank`·상위 % UI는 v3.0에서 **폐기**. 상세 [§13](#13-테스트-석차--게이지).

## §10. 수업기록 (Lesson Record)

| 항목 | 규칙 |
|------|------|
| **정의** | `lesson_date` 기준 **1회 수업** 1건 + `summary`(수업 내용 요약) |
| **귀속** | 숙제·테스트·클리닉·영상 → `lesson_record_id` FK |
| **분리** | 공지·수업 정보·교재 → **수업기록과 별도** |
| **UI** | 반 상세 **수업기록 탭** |

---

## §11. 정오표·채점 (수동)

| 항목 | 규칙 |
|------|------|
| **OCR** | **사용하지 않음** |
| **입력 주체** | **교직원만** |
| **정답지** | 문항별 수동 입력; 반 **모든 조교** 조회 가능 |
| **학생 답안** | 문항별 수동 입력 → 정답지와 비교 |
| **점수** | `raw_score = ceil(100 ÷ question_count × correct_count, 소수 첫째 자리)` |
| **완료** | 학생별 점수 저장 시 완료 |
| **마감** | **`due_at` 없음** |

---

## §12. 재시험

| 항목 | 규칙 |
|------|------|
| **설정** | `retake_threshold_count`(문항 수) |
| **판정** | 학생 점수 **올림**, 기준 점수 **내림** 환산 후 비교 |
| **횟수** | 최대 **3회** |
| **최종 점수** | **마지막 시험**; 통과 시 재시험 대상 제외 |
| **일정** | 선생·조교 등록 → 대상 학생 알림 1회 |
| **석차** | 본·재시험 각각 독립 계산 |

---

## §13. 테스트 석차 · 게이지

| 용도 | 규칙 |
|------|------|
| **UI·보고서** | **석차** (`rank`). ~~상위 %~~ 폐기 |
| **동점** | 공동 등수, 다음 등수 건너뜀 (100·100·99 → 1·1·3) |
| **게이지 30%** | 완료 시험 `raw_score` **산술 평균** ×0.3 |
| **DB** | `rank` 추가. `upper_rank_pct`, `percentile_rank` deprecated |

## §16. 알림 완료 기준

| 원칙 | 규칙 |
|------|------|
| 발송 | **1회** |
| 소멸 | **완료** 시만 (과제형). **읽음 ≠ 소멸** |
| 뱃지 | **미완료 건수** (`pending-count`). 읽음과 무관 |
| D-1 | 숙제 **폐기** · 테스트는 `test_at` **전날** 1회 유지 |

| 유형 | 완료(소멸) | 읽음 처리 |
|------|------------|-----------|
| 숙제 | **점수 입력** | 열람만 (뱃지 유지) |
| 테스트·재시험 | **점수 입력** | 열람만 |
| 영상 | **인증 사진 제출** (대상 학생) | 열람만 |
| 클리닉 | **결과 입력** | 열람만 |
| 공지·배정·보고서·결과 알림 | — | 읽음 시 목록에서 정리 |

| UX | 규칙 |
|----|------|
| 미완료 요약 | "아직 완료하지 않은 과제가 N건 있습니다" + 클릭 시 **미완료 목록** |
| 홈·알림 | 동일 `pending-tasks` 데이터 소스 |
