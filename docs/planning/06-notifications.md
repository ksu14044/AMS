# 06. 알림 시스템

학생 메인 우측 상단 🔔 — 미확인 개수 뱃지.

## 알림 유형

| 유형 | 트리거 | 수신 |
|------|--------|------|
| 공지 등록 | 반 공지 게시 | 해당 반 학생 |
| 영상 등록 | 영상 업로드 | 해당 반 학생 |
| 숙제 등록 | 숙제 예정 등록 | 해당 반 학생 |
| 숙제 D-1 | 마감 전일 09:00 | 해당 반 학생 |
| 테스트 등록 | 테스트 예정 등록 | 해당 반 학생 |
| 테스트 D-1 | 시험 전일 09:00 | 해당 반 학생 |
| 클리닉 확정 | 토 23:00 잠금 후 | 예약 학생 개별 |
| 클리닉 변경 | **선생님이** 내 클리닉 슬롯을 수정 | 해당 학생 개별 |
| 결과 입력 | 숙제/테스트/클리닉 결과 | 해당 학생 개별 |
| 수강반 배정 | 관리자 배정 | 해당 학생 개별 |
| 보고서 생성 | 성실도 보고서 완료 | 해당 학생 개별 |

교직원도 역할에 따라 알림 수신(기획서 매트릭스 ✓).

## 학생 알림 목록 UI (원본 §08)

- 필터: `[ 전체 ]` `[ 미확인 ]`
- 예시:
  - 🔴 [국어 남고1반] 6월 1주차 테스트 내일입니다 · 방금
  - 🟣 [국어 남고1반] 새 공지 등록: 모의고사 대비 안내 · 1시간 전
  - 🟢 [영어 남고2반] 클리닉 예약 확정 (월 19:00) · 어제
  - ⚪ [국어 남고1반] 영상 수업 업로드: 현진건 작품 분석 · 3일 전

## 구현

- 저장: `notification` + `user_notification`
- 발송: FCM (웹 푸시 optional)
- 스케줄: Spring `@Scheduled` — D-1 09:00, 클리닉 잠금
- 도메인 이벤트: `ApplicationEvent` → 리스너에서 알림 생성
