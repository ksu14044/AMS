## §1. 역할 7종 (교직원 가입)

### 원본 근거

- 회원가입: `역할 선택(7종)` (문서 말미)
- §2-2 선생님 4행: 국어·영어·수학·**행정**
- §2-3 조교 3행: 국어·영어·수학
- §2-1 관리자(원장·학원 관계자): **별도 그룹**, 가입 플로우 7종에 포함되지 않음 → 원장은 승인자

### 확정 — 7종 enum

| 코드 | 표시명 | 비고 |
|------|--------|------|
| `TEACHER_KO` | 국어 선생님 | 담임 가능, 과목 KO |
| `TEACHER_EN` | 영어 선생님 | 과목 EN |
| `TEACHER_MATH` | 수학 선생님 | 과목 MATH |
| `STAFF_OFFICE` | 행정 | 학원 전체 보조 |
| `ASSISTANT_KO` | 국어 조교 | |
| `ASSISTANT_EN` | 영어 조교 | |
| `ASSISTANT_MATH` | 수학 조교 | |

**원장·학원 관계자:** `ACADEMY_ADMIN` — 최초 학원 개설 계정 또는 별도 초대.

**교직원 활성화 (§2-1 정합):** 가입 시 **7종 중 역할을 선택**하되, `PENDING` 상태로 등록. **원장(또는 `ACADEMY_ADMIN`) 승인 시** 역할·과목을 **확정 또는 수정**한 뒤 `ACTIVE`. §2-1 「선생님 회원가입 시 역할·과목을 지정」= **승인 단계에서 관리자가 최종 확정**하는 것으로 해석한다.

**학생:** `STUDENT` — 별도 가입 플로우 (7종 아님).

## §2. 행정 — 공지 범위

### 원본 근거

- 행정: `학원 전체 운영 보조, 공지 등록` (§2-2)
- 반 공지: 담임/조교는 **반 단위** (§04, §06-2)

### 확정

| 공지 종류 | 작성 | 노출 |
|-----------|------|------|
| **학원 공지** (`academy_notice`) | `STAFF_OFFICE`, `ACADEMY_ADMIN` | 학원 소속 전체(또는 대시보드) |
| **반 공지** (`class_notice`) | 담임·조교(담당 반), 관리자 | 해당 반 수강생 |

행정의 「공지 등록」= **학원 공지** 우선. 반 공지는 담당 반 없으므로 기본은 학원 공지, 필요 시 관리자가 반 공지도 가능(매트릭스 ✓).

## §3. 「출석」 (학생 목록 탭)

### 원본 근거

- §5-2 학생 목록: `수강 학생 명단·출석·공부기록 확인`
- §11-1·클리닉: **클리닉 출석** 여부·횟수는 명확
- **수업(화·목 수업) 출석** 별도 입력·테이블 **없음**

### 확정

- **별도 `class_attendance`(수업 출석부) 테이블은 MVP 제외.**
- 학생 목록 탭 「출석」= **클리닉 출석 요약 + 공부기록 클리닉 출석률** 조회.
- Phase 2 이후 필요 시 수업 출석 테이블 추가.

## §5. 숙제·테스트 — 학생별 결과

### 원본 근거

- §11-1 숙제: `제출 여부, 제출 일시, 결과(점수/등급)` — **학생 단위**
- 보고서 예시: `7 / 8건`, `84점 / 반 평균 78점`, `상위 23%`
- §04 숙제/테스트: 반 단위 **과제 정의** + 완료 시 **결과**

### 확정

| 레이어 | 설명 |
|--------|------|
| `homework` / `test` | 반 단위 과제·시험 **정의**(예정/완료) |
| `homework_submission` | 학생별 제출 여부·일시·점수/등급·메모 |
| `test_score` | 학생별 점수·등급; 반 평균·상위 %는 시험 단위로 계산 저장 |

교사/조교가 **학생별 결과 입력** API 제공. 학생은 조회 + (§8) 제출 상태는 교사 입력 또는 추후 확장.

## §7. 영상 인증 — 가산 반영

### 원본 근거

- §11-1: 영상 인증 `가산 반영` (가중치 표에는 40/30/30만 명시)
- 보고서 예시 종합식: `숙제×0.4 + 클리닉×0.3 + 테스트×0.3` — **영상은 항목별 등급만, 종합 점수 식에 미포함**

### 확정

| 지표 | 계산 |
|------|------|
| **종합 달성률(게이지)** | 숙제 제출률×0.4 + 클리닉 출석률×0.3 + 테스트 `percentile_rank` 평균×0.3 ([§9](#9-테스트-점수--종합-게이지-30-환산)) |
| **영상 인증률** | 별도 % · 공부기록 화면·보고서에 표시 |
| **가산** | 종합 점수에 **가산하지 않음** (원본 보고서 예시와 동일). 「가산 반영」= 보고서·기록에 **별도 항목으로 반영** |

### 원본 근거

- 학생 반 홈: `완료: … [제출 완료]` (표시)
- §03: 학생 숙제 등록/결과 입력 **✗**

|------|------|

| 추후 | 학생 제출 기능은 v2 (기획서 범위 외 명시 없음) |

## §9. 테스트 점수 — 종합 게이지 30% 환산

### 원본 근거

- §11-1: 테스트 `응시 여부, 점수, 등급, 반 평균 대비` — 가중치 **30%**
- 보고서·UI: `84점 / 반 평균 78점`, `상위 23%` (표시용)
- 종합 게이지 식: `숙제×0.4 + 클리닉×0.3 + 테스트×0.3` — **원점수를 그대로 넣는 규칙은 없음**

### 확정

| 용도 | 규칙 |
|------|------|
| **DB 저장** | `test_score`: `raw_score`, `class_avg`, `upper_rank_pct`(상위 %, 예: 23), `percentile_rank`(0~100, **높을수록 우수**) |
| **percentile_rank 계산** | 해당 `test`·반 수강생 중 `raw_score`가 있는 학생만 대상. 동점은 동순위. `percentile_rank = (본인보다 낮은 점수 인원 수 / (응시 인원 − 1)) × 100` (응시 1명이면 100) |
| **상위 % 표시** | `upper_rank_pct = 100 − percentile_rank` (반올림 정수). 보고서·UI는 이 필드 사용 |
| **종합 게이지 테스트 항목** | 집계 기간 내 **완료된 시험**마다 `percentile_rank`를 구한 뒤 **산술 평균** → ×0.3. 완료 시험 0건이면 테스트 항목 = 0 |
| **미응시** | 해당 시험의 `percentile_rank = 0`, 평균 분모에는 **포함** |
| **성실도 보고서 종합 테스트 30%** | `raw_score`를 만점 100 기준 0~100%로 환산 후 ×0.3 (공부기록 게이지와 **별도**) |
| **보고서 표시** | 점수·반 평균·**상위 %** (`upper_rank_pct`) · `percentile_rank`는 DB 저장·참고. 종합·테스트 등급·막대 %는 **원점수 %** |
