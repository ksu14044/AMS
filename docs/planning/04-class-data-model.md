# 04. 반(Class) 데이터 구조

각 반은 **7개 섹션**으로 구성. 선생님/조교 편집, 학생은 수강 반 **읽기 전용**.

| 섹션 | 주요 필드 | 편집 주체 | 비고 (원본 §04) |
|------|-----------|-----------|------------------|
| 수업 정보 | 요일, 시간, 강의실 | 담임 선생님 | |
| 공지 | 제목, 내용, **첨부파일**, 작성일 | 선생님 / 조교 | |
| 교재 | 책 제목, 출판사, 현재 진도 | **담임 선생님** | |
| 영상 수업 | 유튜브 링크, **썸네일 자동**, 제목, 날짜 | 선생님 / 조교 | 학생 인증사진 첨부 가능 |
| 숙제 | **완료:** 제목+결과 / **예정:** 제목+일시 | 선생님 / 조교 | 결과 입력 가능 |
| 테스트 | **완료:** 제목+결과 / **예정:** 제목+일시 | 선생님 / 조교 | 결과 입력, 시험 후 보고서 트리거 |
| 클리닉 | 세션 슬롯, 담당 조교, 학생 예약 상태, 결과 | 선생님 생성 / 학생 예약 | 조교가 클리닉 결과 입력 가능 |

교재 편집 권한·조교 조회만: [DECISIONS.md](./DECISIONS.md) §4.

## 엔티티 관계(개념)

```
Academy 1──* Class 1──* [Notice, Textbook, Video, Homework, Test, ClinicSlot]
                │
                *──* Student (via class_enrollment)
                │
                └── AssistantAssignment (조교)
```

## 상태값 예시

- 숙제/테스트: `SCHEDULED` | `COMPLETED`
- 클리닉 슬롯(주차): `OPEN` | `LOCKED`
- 클리닉 예약: `EMPTY` | `RESERVED` | `CONFIRMED`
- 영상 인증: `PENDING` | `SUBMITTED`

상세 테이블명은 [database/schema-overview.md](./database/schema-overview.md).
